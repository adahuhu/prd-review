## 一、模板结构速查

### B 型（综合优化）

```
需求标题名称（优先级）
├── 1.1 背景
│   ├── 现状与问题：当前功能表现 + 影响
│   ├── 需求来源：客户/场景（若有）
│   ├── 优化内容：整体方案
│   └── 影响点：功能点超过两项时列出
├── 1.2 功能点一
│   ├── 方案 + 详细描述
│   └── 交互变更说明：Before → After（按需）
├── 1.3 功能点二（按需）
├── 1.4 影响范围（若不涉及可删除）
└── 2. 需求标题名称二（聚合文档时继续编号）
```
- 篇幅：1-2 页
- 支持：单个需求 + 多个需求聚合

### A 型（新功能）

```
需求名称（头部：目标上线时间 / 原型图 / 设计稿）
├── 1. 背景与目标（业务背景 / 目标用户 / 业务目标）
├── 2. 用户场景（角色表 / 典型场景）
├── 3. 竞品分析（有参考时写，无参考删除）
├── 4. 业务流程
│   ├── 4.1 流程图（主流程 + 分支 + 异常 + 回滚）
│   ├── 4.2 状态机（涉及状态流转时必填）
│   └── 4.3 字段定义（新增核心字段时填写，无则删除）
├── 5. 功能描述
│   ├── 5.1 功能清单（功能项超过 5 点时启用）
│   └── 5.2 各功能点
├── 6. 权限（功能权限 / 数据权限 / 字段权限）
└── 7. 其他模块（按需：非功能需求/影响范围/跨系统集成/数据更新频率/数据迁移/移动端）
```
- 篇幅：8-15 页

### C 型（临时紧急）

```
紧急需求（YYYY-MM-DD）
├── 1. 标题名称
│   ├── 现状与问题 + 影响
│   ├── 触发条件（复现路径或触发前提）
│   ├── 来源（哪个客户）
│   └── 目标上线时间（具体时间点）
├── 1.1 优化内容
│   ├── 方案描述
│   └── 具体改动点
└── 1.2 影响范围
    ├── 涉及模块 + 影响说明
    ├── 存量数据处理
    ├── 是否有后续正式需求
    └── 其他影响
```
- 篇幅：<1 页
- 优先级默认 P0

## 二、类型判断规则

```
不在迭代内 + 紧急 → C 型
↓ 不是
新增交互流程 或 跨系统交互 → A 型
↓ 不是
其他 → B 型
↓ 拿不准
用 A 型
```

## 三、写作规范

| 规则 | ✅ 正确做法 | ❌ 错误做法 |
|------|-----------|-----------|
| 动词开头 | "创建采购订单""查询汇率列表" | "采购订单的新建""汇率的查看" |
| 主语明确 | "用户点击提交后，系统校验必填字段" | "点击后校验" |
| 可量化 | "支持 USD、EUR、GBP、JPY、CNY 五种币种" | "支持多币种" |
| 无歧义 | "汇率每日 00:00（UTC+8）从 OANDA 获取" | "汇率自动更新" |

**模糊词黑名单**：适当、合理、相关、若干、等、类似、尽量、必要时、优化、完善

## 四、各类型审核重点

### B 型重点
- 现状与问题必须写清：当前什么表现 + 产生什么影响
- 功能点有方案 + 详细描述
- 影响范围已评估（含存量数据）— **注意：B 型影响范围为建议项，检测到涉及跨模块/跨系统/存量数据时提示补充，不扣分**
- 允许 Before→After 对比表

### A 型重点
- 背景与目标清晰，业务目标尽量量化
- 用户场景有角色表和典型使用场景
- 竞品分析：有参考写完、无参考删除（不留空）
- 业务流程覆盖主流程 + 分支 + 异常 + 回滚
- 状态机采用**操作驱动型**：当前状态→执行操作→流转后状态→角色限制→说明
- 字段定义：字段/业务含义/数据来源/更新时间
- 功能清单只在功能项 >5 个时启用
- 每个功能点含：功能描述 / 前置条件 / 页面流程 / 业务规则 / 异常场景

### C 型重点
- 四个必填：现状与问题 / 触发条件 / 来源 / 目标上线时间
- 触发条件帮助研发定位，无法复现时注明
- 目标上线时间是具体时间点，不写"尽快"
- 影响范围四维：涉及模块 / 存量数据 / 是否有后续正式需求 / 其他影响
- **不要求**：竞品分析、状态机、用户场景

## 五、A 型第 7 章（其他模块）按需判断

| 模块 | 触发条件 |
|------|----------|
| 非功能需求 | 有性能/安全/权限特殊要求 |
| 影响范围 | 涉及其他模块的业务/数据影响 |
| 跨系统集成 | 涉及 ERP/WMS/SCM/LPS 间数据交互 |
| 数据更新频率 | 涉及定时拉取/同步外部数据 |
| 数据迁移与兼容 | 涉及存量数据处理或接口变更 |
| 移动端配套说明 | 需求含手机端/移动审批/移动查询功能 |
